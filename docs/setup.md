# セットアップ詳細

## 1. アプリを起動

```bash
git clone https://github.com/kento-cell/slide-forge
cd slide-forge
npm install
npm run dev
# → http://localhost:5173
```

## 2. AI モードを選ぶ

初回起動時にウィザードが開きます。3 つから 1 つ選択。

### A. クラウド AI (推奨: Gemini)

1. ウィザードで「☁ クラウド AI」を選択
2. プロバイダタブから **Google Gemini** を選択
3. [🌐 キー取得ページを開く] → Google AI Studio で API キー作成 (無料)
4. キーを貼り付け → [接続テスト] → ✅ → [完了]

無料枠: Gemini Flash は 1 日 1,500 リクエストまで無料 (個人用途では事実上無制限)。

### B. ローカル AI (Ollama)

1. ウィザードで「💻 ローカル AI」を選択
2. Ollama 未インストールなら [🌐 Ollama をダウンロード] (~200MB)
3. Ollama インストール後 [再検出] → ✅
4. モデル選択 (推奨: **Qwen 2.5 14B** / 軽量 PC は **Llama 3.1 8B**)
5. [📥 ダウンロード開始] (5〜10 分、PC 内に保存)
6. [接続テスト] → ✅ → [完了]

完全オフラインで動作。データは外部送信なし。

### C. オフライン (AI なし)

ウィザードで「✏ AI なし」を選択するだけ。Markdown を機械的に PPTX に変換します。

## 3. プロンプトを書く

メイン画面の大きなテキストエリアに:

- **そのまま [▶ 生成]** — 灰色サンプル文がそのまま使われる (動作確認に最適)
- **編集して [▶ 生成]** — あなたのプロンプトで生成
- **`.md` `.txt` をドロップ** — ファイルから読み込み

## 4. 結果を DL

- スライドサムネを確認
- [⬇ PPTX をダウンロード] でローカル保存
- [♻ 再生成] で別パターンを試す
- [Markdown を見る] で AI が出力した md を確認

## トラブルシュート

| 症状 | 対処 |
|---|---|
| 接続テストが失敗する | API キーの先頭/末尾の空白を削除 / モデル名を変更 |
| Ollama 検出しない | `ollama serve` でサービス起動を確認 / `http://localhost:11434/api/version` をブラウザで開いて応答確認 |
| Anthropic で CORS エラー | ブラウザコンソールのエラー文確認 / 一旦 Gemini や Groq に切替 |
| 生成された md が壊れている | プロンプトに「Markdown のみで、コードフェンス無しで出力」を加える |
| 「スライドを抽出できませんでした」 | プロンプト末尾に「`# タイトル` と `## 見出し` を必ず使う」を追記 |

## 設定の保存場所

ブラウザの localStorage (`slide-forge.settings.v1`)。
- API キー: 平文で保存 (将来 Tauri Keyring へ)
- 別ブラウザ / シークレットモードでは別設定として扱われる
